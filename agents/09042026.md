# Agents — 09 Apr 2026

## What I did
- Reviewed a maze-bench run and identified a benchmark integrity failure: a GPT Codex agent inspected local maze source/state files to infer the maze layout instead of navigating from the rendered first-person PNG view.
- Collected and organized evidence for the failure mode:
  - incident recording from the run
  - additional screenshot showing the agent explicitly giving up when it could not continue under the intended navigation-only constraint
- Turned the finding into a concrete benchmark fix rather than leaving it as a one-off incident note.
- Updated the maze skill policy to make the allowed information boundary explicit.
- Added a large direction arrow to the maze right toolbar so the robot's heading is easier to read; the agent got confused with the direction, so the arrow reduces complexity and makes orientation more obvious.
- Opened and documented the remediation PR:
  - **Pull Request:** https://github.com/menloresearch/agent-bench/pull/3
  - **Branch:** `fix/maze-bench-no-source-inspection-requirement`

## Findings / results
- The benchmark as originally written left enough ambiguity that an agent could inspect implementation/state artifacts and still appear to complete the task.
- That behavior invalidates maze-bench as a visual-navigation benchmark because the agent is no longer solving from observation + action alone.
- The run also exposed a second useful signal: when restricted to the intended interface, the agent later reported that it was stuck and could not continue without guessing or violating the rules.
- Main takeaway: benchmark integrity constraints need to be treated as first-class benchmark design, not as an afterthought.

## Remediation implemented
- Added a **Benchmark integrity requirement** to `agent-bench/benchmarks/maze-bench/skills/maze-game/SKILL.md`.
- The new rule explicitly forbids reading maze source/state artifacts (for example `maze_game.py`, `score_maze_run.py`, or state files) to derive layout or path information.
- The intended solving interface is now stated more clearly: agents must rely only on rendered views plus allowed CLI commands.
