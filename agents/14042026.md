## Findings / results
- `agent-bench` now expands beyond single-agent maze navigation into **multi-agent collaboration**.
- The benchmark distributes information asymmetrically across 3 roles:
  - **Cartographer** sees global topology
  - **Scavenger** is the only role that can move and collect resources
  - **Engineer** knows the exact beacon blueprint and required materials
- This creates a useful autonomy test for:
  - fragmented-information reasoning
  - communication quality
  - role discipline / interface constraints
  - multi-step planning with execution delegated to one acting role
- The same benchmark-integrity lesson from maze-bench was carried over: the skill explicitly forbids reading source/state files such as `island_game.py`, `score_island_run.py`, `islands.py`, and `island_state.json`.

## Why this matters
- The project is no longer only about whether an agent can interpret a rendered world and act inside it.
- It now also tests whether multiple agents can coordinate under partial observability using only allowed interfaces.
- That is closer to real robotic-autonomy settings where different subsystems or robots hold different pieces of the world state and must communicate to complete a task.
